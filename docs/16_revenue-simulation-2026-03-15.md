### 前提条件
- CLI 全機能無料（scan -g 全フォーマット + MCP + watch）

## 3. シナリオB: PMF分析推奨プラン

### 前提条件
- **月1-4**: Custom Rules Engine 実装 + 手動セクション保護 + フリーミアム再設計
- **月2-3**: npm publish + ローンチ（Custom Rules 付きで差別化訴求）
- **月3-6**: Design Partner Program（10チーム無料提供 → フィードバック → 改善）
- **Free は CLAUDE.md のみ**、マルチフォーマット + Custom Rules は Pro
- **Pro: $12/月** / **Team: $20/user/月**（実装済み価格を維持）
- **Free→Pro転換率: 3-5%**（Custom Rules が刺さる前提）
- **Team**: Design Partner 経由で月6以降徐々に獲得
- **月次チャーン率: 5%**（Custom Rules でロックイン効果あり）

### npm DL 成長モデル

- Custom Rules は Repomix にない独自機能 → 差別化で検索流入増加
- 月次 15-20% 成長（開発者ツールの典型的オーガニック成長率）

## 10. 推奨事項と結論

### 推奨: シナリオC（最速収益化）を採用し、以下の優先順位で実行

#### 即座に実行（今週中）
1. **Lifetime Deal の準備**: Stripe で $149 の one-time product を作成。50枠限定。LPを作成
2. **Custom Rules MVP（2週間）**: `.orbit/rules.yml` の基本構造だけ実装。完璧を目指さない
3. **フリーミアム再設計**: Free を CLAUDE.md のみに制限するフラグを追加

#### 月1-2
4. **Paid Beta ローンチ**: Custom Rules MVP + Lifetime Deal を同時発売
5. **Product Hunt + Hacker News ローンチ**: 「Repomix meets team coding standards」で訴求
6. **Build in Public**: X で毎日進捗を共有。MRR を公開

#### 月3-4
7. **CI/CD GitHub Action 公開**: Team プラン訴求の起爆剤
8. **Design Partner Program 開始**: 10チームに無料提供。フィードバック収集
9. **コンサルティング開始**: 「CLAUDE.md 最適化セッション」を $200 で提供

### なぜシナリオCか

| 理由 | 説明 |
|------|------|
| **初期キャッシュ** | LTD $7,450 で12ヶ月分以上のサーバー費を即座に確保 |
| **心理的安全** | 赤字期間ゼロで開発に集中できる |
| **早期フィードバック** | Paid Beta ユーザーは無料ユーザーより質の高いフィードバックを提供 |
| **FOMO効果** | 「限定50枠」「ベータ価格は今だけ」で初期転換率を押し上げ |
| **Build in Public** | MRR公開がそのままマーケティングになる（開発者コミュニティで有効） |

### リスク要因
| リスク | 確率 | 影響 | 対策 |
|--------|------|------|------|
| Anthropic が CLAUDE.md 自動生成を内製 | **高** | 致命的 | Custom Rules でロックイン。ポータビリティが価値 |
| LTD 購入者がサポート負荷に | 中 | 中 | Discord コミュニティで self-serve |
| Custom Rules の開発が4週間超過 | 中 | MRR遅延 | MVP を2週間で出し、イテレーション |
| npm DL が想定の半分 | 中 | MRR 半減 | コンテンツマーケ強化、OSS 貢献で認知向上 |
