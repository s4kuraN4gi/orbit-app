### PMF分析からの核心的制約

**「5人全員が現状では払わない」という事実を数値に織り込む:**

- Free CLI（scan -g全フォーマット + MCP + watch）で価値が完結 → 転換動機なし
- Custom Rules Engine がない = Pro への転換率は理論上 0.3-1% の天井
- Team プランは個人 PMF なしには売れない（PMF分析の総意）
- Anthropic/Cursor による内製化リスクは「高」確率 — 12ヶ月以内に現実化する可能性あり

> **0.5x の意味**: Custom Rules が期待ほど刺さらない。「ちょっと便利」止まり。Dev.to 記事の反響が薄い。
> **2x の意味**: Custom Rules が鎮痛剤として強く機能。DP のUX 証言が口コミを生む。

#### シナリオC（ベース転換率: 5-7%）

| ケース | 転換率 | 月12 Pro 数 | 月12 MRR（Pro+Team） |
|--------|--------|------------|-------------------|
| 転換率 0.5x（低迷） | 2.5-3.5% | 約60人 | 約$1,380 |
| ベースケース | 5-7% | 120人 | $2,448 |
| 転換率 2x（好調） | 10-14% | 約240人 | 約$4,200+ |

> **シナリオCの 0.5x でも MRR $1,380** → 個人開発として十分成立する。シナリオCの基本戦略は堅牢。

### チャーン率感度

| シナリオ | チャーン 0.5x | ベース | チャーン 2x |
|---------|-------------|--------|-----------|
| B（ベース5%） | MRR ~$780（+18%） | $662 | MRR ~$490（-26%） |
| C（ベース3.5%） | MRR ~$2,750（+12%） | $2,448 | MRR ~$1,980（-19%） |

> **重要**: Custom Rules のロックイン品質 = チャーン率への最大の影響変数。`.orbit/rules.yml` がプロジェクトにコミットされれば、ツール乗り換えコストが発生する（良いロックイン）。

### npm DL 感度（マーケティング実行品質）

| ケース | 月12 DL/週（シナリオB） | 月12 MRR（シナリオB） |
|--------|----------------------|---------------------|
| DL 0.5x（コンテンツ不発） | 900 | ~$350 |
| ベースケース | 1,800 | $662 |
| DL 2x（バイラル効果） | 3,600 | ~$1,200 |

**認知の獲得が最大の不確実性。** 技術品質が高くても、Dev.to/Zenn 記事の品質と配信が MRR を 3-4x 変動させる。

## 7. クリティカルマイルストーンと Go/No-Go 判定

### マイルストーン一覧

| # | 時期 | マイルストーン | 判定基準 |
|---|------|--------------|---------|
| M1 | 月1-2 | Custom Rules MVP リリース | `orbit rules add` + YAML 形式での保存が動作すること |
| M2 | 月2 | npm publish + Product Hunt ローンチ | PH: 100 upvote以上 / npm 初週 500 DL以上 |
| M3 | 月3 | 最初の有料転換（Pro 1人） | **$12 が Stripe に入金されること** |
| M4 | 月4 | GitHub Actions 公開 | GitHub Marketplace 承認 + 10 repo で使用 |
| M5 | 月5 | Design Partner 5人以上アクティブ | 週次でフィードバックを受け取れている状態 |
| M6 | 月6 | MRR $100 達成 | **Pro 9人 または Team 1チーム + Pro 5人** |
| M7 | 月6 | Sean Ellis テスト実施 | DP 10人以上にアンケート実施 |
| M8 | 月9 | MRR $500 達成（シナリオCのみ） | 継続的な月次成長の確認 |
| M9 | 月12 | MRR $1,000 達成（シナリオC） | ARR $12,000 = 持続可能な個人開発 |

### Go/No-Go 判定ポイント

#### 判定ポイント1: 月3終了時

**Go 条件（全て満たすこと）:**
- Pro ユーザー 1人以上（有料転換の実証）
- npm DL/週 300 以上（成長軌道の確認）
- Custom Rules を実際に使っているユーザーが1人以上

**No-Go → ピボット検討:**
- 月3でPro ユーザー 0人 かつ 転換の見込みがない
- Custom Rules の開発が遅延し月4以降にずれ込む
- npmDL が週100以下（認知獲得の根本的問題）

**No-Go 時のアクション:** フリーミアム設計の再検討。Free をさらに絞る（CLAUDE.md の行数制限等）。あるいは完全に方向転換してコンサルティング主軸にシフト。

---

#### 判定ポイント2: 月6終了時

**Go 条件（継続）:**
- MRR $100 以上
- 月次チャーン率 8% 未満
- Sean Ellis "Very disappointed" 20% 以上（10人中2人）
- DP 少なくとも 5チームがアクティブ

**No-Go → 重大ピボット判断:**
- MRR $50 未満（固定費すら出ない）
- Sean Ellis "Very disappointed" 0%（誰も不可欠と感じていない）
- Custom Rules を実際に使っているPro が 50% 未満

**No-Go 時のアクション:**
1. タスク管理機能を完全削除し、CLI + Custom Rules 専業にする
2. Team 機能への投資を中断
3. 価格を $6/月に下げ、低価格で転換率を稼ぐ
4. あるいは SaaS を停止し CLI を完全 OSS 化（GitHub Sponsors + コンサルで収益化）

---

#### 判定ポイント3: 月9終了時

**Go 条件（フル実行継続）:**
- MRR $500 以上（シナリオBなら月12で達成予測）
- Pro チャーン率が安定（月次 5% 以下）
- Team が 3チーム以上（チームセールスの実証）
- Anthropic/Cursor による内製化が発表されていない

**No-Go → 方向転換:**
- MRR $200 未満で頭打ち感
- Anthropic が CLAUDE.md 自動生成を正式発表 → Custom Rules の価値が著しく下がる
- Pro チャーン率 10% 超（有料機能の価値不足）

**No-Go 時のアクション（プラットフォーム内製化の場合）:**
- **B2B 特化にピボット**: 個人開発者向けは捨て、「エンタープライズ向けコーディング標準管理」に転換
- **ContextIR オープン仕様化**: 標準化団体として認知を得る
- **CLI → ライブラリ化**: 他ツールに組み込まれる SDK として提供

|--------|------|--------|---------|------|

## 9. 推奨アクション

### 最終推奨: **シナリオBをベースに、シナリオCの施策を選択的に追加**

**根拠:**
- シナリオBは現実的で実行可能。1人での並行作業に見合った負荷
- シナリオCの全施策（カンファ登壇 + バイラル記事 + モノレポ対応 + Python対応）を同時実行するのは現実的でない
- **「シナリオBを確実にやり切り、余力でシナリオCの施策を追加する」** が最適解
