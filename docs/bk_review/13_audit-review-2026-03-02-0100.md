### Critical（致命的）

#### C-UX-1. ダッシュボードがモバイルで完全に使用不能

対象: `components/dashboard/DashboardView.tsx` L183-229

ヘッダーバーに 7 つのボタンが横一列に並び、レスポンシブブレークポイントが一切ない。768px 以下ではボタンが画面外にはみ出すか横スクロールが発生。タッチターゲットが密集して誤タップ頻発。

修正方針: ハンバーガーメニューまたは `...` ドロップダウンにアクション群をまとめ、主要アクション（New Task）のみモバイルで常時表示。

#### C-UX-2. `dateGroups` i18n キーが完全に欠落

対象: `components/dashboard/TaskList.tsx` L77, L529

`useTranslations('dateGroups')` を呼んでいるが、`messages/en.json` / `messages/ja.json` のどちらにも `dateGroups` セクションが存在しない。TaskList のグループヘッダーに `overdue`, `today`, `tomorrow` 等の生キー名が表示される。

修正方針: 両言語ファイルに `dateGroups` セクションを追加:
```json
"dateGroups": {
  "overdue": "Overdue",
  "today": "Today",
  "tomorrow": "Tomorrow",
  "thisWeek": "This Week",
  "nextWeek": "Next Week",
  "later": "Later",
  "noDate": "No Date"
}
```

#### C-UX-3. ログインページが完全に i18n 未対応

対象: `app/login/page.tsx` 全体

`useTranslations()` が一切使われておらず、全テキストがハードコード英語。`auth` キーは両言語ファイルに既に存在するにもかかわらず未使用。日本語設定でもログイン画面だけ英語になる。

修正方針: `useTranslations('auth')` を導入し全テキストを i18n キーに置換。

### Critical（致命的）

#### C-ARCH-1. Dashboard の scanData 全量フェッチ（OOM リスク）

対象: `app/dashboard/page.tsx` L83-93

`projects.scanData`（JSONB、最大 1MB）を全プロジェクト分 SELECT。プロジェクト 50 個で 50MB のメモリ消費。Vercel の 1024MB 制限で OOM の可能性。

修正方針: プロジェクト一覧では `scanData` を除外し、`currentProject` 分のみ別クエリで取得。

#### C-ARCH-2. スナップショット削除の O(n) 個別 DELETE ループ

対象: `app/api/cli/projects/[id]/scan/route.ts` L89-100

50 件超のスナップショットを 1 件ずつ直列 DELETE。Neon HTTP driver では各 DELETE が独立 HTTP リクエスト。100 件で 50 回のラウンドトリップ。

修正方針: サブクエリで 1 クエリに統合:
```sql
DELETE FROM scan_snapshots
WHERE id NOT IN (SELECT id FROM scan_snapshots WHERE project_id = ? ORDER BY created_at DESC LIMIT 50)
```

#### C-ARCH-3. bulk-create の直列 INSERT（トランザクションなし）

対象: `app/api/tasks/bulk-create/route.ts` L16-51

200 タスクを 1 つずつ直列 INSERT。トランザクションなしのため、途中エラーでデータ不整合。Vercel の 10 秒タイムアウトに到達する可能性。

修正方針: フラット化して一括 INSERT + トランザクションで囲む。

## 結論

**プロダクト/UX (4.5/10)**: i18n 基盤は 500+ キーが整っているのに、適用漏れが多すぎる。ログインページの完全ハードコード、TaskList の dateGroups 欠落は品質管理の問題。モバイル対応はダッシュボードが完全に壊れている状態で、2026 年のプロダクトとしては許容できない。逆にデスクトップ UX（キーボードナビ、D&D、ImportPlanModal）は確実に良い。

**アーキテクチャ/パフォーマンス (5.0/10)**: ContextIR パイプライン、Stripe 統合、セキュリティヘッダーは高品質。しかし Dashboard の scanData 全量フェッチ（OOM リスク）、bulk-create のトランザクション不在、O(n) DELETE ループは、ユーザー増加で確実に顕在化する時限爆弾。DRY 違反（requireUser 3 重定義、formatTask 3 重定義）は保守性を大きく損なう。

**優先順位**: Critical 5 件（3-4h）を即座に修正し、High 9 件（8-10h）を今週中に完了すれば、両領域とも 6.5-7.0 に到達可能。
