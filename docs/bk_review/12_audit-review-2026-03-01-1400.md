### 致命的欠陥: CLI ゲートのバイパス

**`packages/cli/src/lib/plan-check.ts` L13-16:**

```typescript
if (!(await isLoggedIn())) {
  return { allowed: true };  // ← 未認証ユーザーは全機能アクセス可能
}
```

**影響**: ログインしていないユーザーは CLI のすべての Pro 機能を無制限に使用可能。

| 機能 | Free（設定値） | 未ログイン（実際） |
|------|---------------|-------------------|
| `orbit watch` | 0（ブロック） | **無制限** |
| `orbit scan --smart` | 0（ブロック） | **無制限** |
| `orbit plan` | 0（ブロック） | **無制限**（※別途認証あり） |
| `orbit scan --focus` | 1回/月 | **無制限** |
| `orbit scan --issues` | 1回/月 | **無制限** |

> `orbit plan` のみ `requireAuth()` を先に呼んでいるため実質的にはゲートが効いている。しかし他の機能は `checkFeatureAccess()` → `isLoggedIn() === false` → `allowed: true` となりバイパスされる。

**修正方針**: 未ログイン時は `{ allowed: false, message: "Login required..." }` を返すべき。

### Pricing 構造の問題

| 項目 | 現在 | 問題点 |
|------|------|--------|
| Pro | $12/月 | 競合（Repomix 無料、GitHub Copilot $10）対比で割高 |
| Team | $15/ユーザー/月 | Pro +$3 のみ、エンタープライズ差別化が弱い |
| Free → Pro 壁 | 1プロジェクト / 20タスク | 制限が厳しすぎ or 緩すぎ（タスクは使わない可能性） |

**$1,000 MRR 到達に必要な規模**:
- Pro のみ: 84 ユーザー
- Team のみ: 67 ユーザー（各1人換算）
- 現実的な変換率（2-3%）で 200-300 月間サインアップ → **$48-108 MRR が上限**

## 結論

Phase 0〜3 の41タスク完了により、セキュリティは **3.5 → 7.0** と大幅に改善。特に Critical だった認可バイパス・課金操作脆弱性は完全に解消された。

一方、マネタイゼーションは **3.0 → 5.0** と改善幅が小さい。CLI ゲートのインフラは整ったが、`plan-check.ts` の致命的バイパス（未ログイン = 全機能開放）により実質的にゲートが機能していない。これは **15分で修正可能** な問題であり、最優先で対応すべき。

LP の CTA 構造（CLI を Primary に誘導）と Pro 機能の価値訴求（ヒューリスティックを「AI」と表現）も収益化の大きなボトルネックとなっている。
