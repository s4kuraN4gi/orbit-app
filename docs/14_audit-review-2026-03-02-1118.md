### 新規指摘事項

#### Critical

##### N-C-1: DiffSummary のハードコード英語

対象: `components/dashboard/ContextDiffView.tsx` L188-194

```tsx
<span className="font-medium">{diffs.length} changes</span>
{added > 0 && <span>+{added} added</span>}
{removed > 0 && <span>-{removed} removed</span>}
{changed > 0 && <span>~{changed} changed</span>}
```

`DiffBadge` コンポーネント（L149-152）は `t(type)` で正しく i18n 化されているが、`DiffSummary` は翻訳関数を受け取らずリテラル英語を出力。Pro プランの有料機能（Context Diff）で日本語ユーザーに混在言語を見せるのは信頼を損なう。

##### N-C-2: bulk-create が実質トランザクション不在

対象: `app/api/tasks/bulk-create/route.ts`

前回 C-ARCH-3 で「batch 化」とされたが、3 ステップ（AI Context INSERT → タスク INSERT → parentId UPDATE）は `db.transaction()` で包まれていない。ステップ 3 が失敗するとタスクは存在するが親子関係が壊れた状態で DB に残る。200 タスク一括インポート機能でデータ不整合は致命的。

##### N-C-3: IdeaBox サイドバーがモバイルで画面を破壊

対象: `components/dashboard/DashboardView.tsx` L347-351

```tsx
{isIdeaBoxOpen && (
  <div className="w-80 shrink-0">
    <IdeaBox ... />
  </div>
)}
```

固定幅 `w-80`（320px）のサイドバー。モバイル（<640px）で開くとメインコンテンツが潰れるか水平スクロールが発生。モバイル DropdownMenu から Ideas を開けるため、ユーザーが実際に遭遇する問題。モーダルまたはスライドオーバーに変更すべき。

### ポジティブな点

1. **認証・認可アーキテクチャの統一**: `auth-helpers.ts` + `project-access.ts` の 2 層構造。Server Actions と CLI API routes が同一の `verifyProjectAccess()` を共有。LEFT JOIN 1 クエリで個人/チーム認可を完結。テスト 18 件で検証済み。
2. **テストカバレッジの飛躍的改善**: 前回 ~6% → 250 テスト。renderers (64件), validations (39件), context-ir (36件), detector (24件) は網羅的でエッジケースもカバー。
3. **Zod バリデーション体系**: `lib/validations.ts` に全 API スキーマ集約。CLI API routes は全て Zod で保護。テスト 39 件で全スキーマの正常/異常パスを検証。
4. **ContextIR 設計**: `ScanResult → ContextIR → Renderer` のコンパイラ IR パターン。関心の分離が適切。Cursor MDC マルチファイル出力もサポート。
5. **Smart Context アルゴリズム**: 隣接リスト + サフィックスマップで O(V+E) グラフ構築。git diff/commits/branch からのキーワード抽出 + 重み付けスコアリング。
6. **Stripe 課金の堅牢性**: idempotency, signature verification, alert webhook, personal/org サブスクリプション分離, `updateOrgSeatCount` 自動 quantity 更新。
7. **DB スキーマ設計**: 適切なインデックス群、CASCADE/SET NULL の FK、unique 制約、usage テーブルの UPSERT パターン。
8. **レート制限**: 全 CLI API route に適用。Upstash Redis（本番）+ インメモリ（開発）のフォールバック。sliding window。
9. **セキュリティヘッダー**: HSTS, X-Content-Type-Options, X-Frame-Options, Permissions-Policy, Referrer-Policy 完備。
10. **アラート基盤**: Slack/Discord webhook への構造化エラー通知。Stripe エラー時に自動通知。
